# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

An "emulator competition" project distributed as the `cmbemu` Python package. Participants write neural-network emulators for the CMB power spectra (TT, TE, EE, $\phi\phi$) and are scored on joint precision + CPU inference speed against a fixed training/test set generated by `classy_szfast`.

**Scope constraint:** the only cosmology code used here is `classy_sz` (`classy_szfast` backend, `cosmo_model=6` = `ede-v2`). Not CLASS, CAMB, or MontePython.

## Data

- **Hosted on HF Hub** at [`borisbolliet/cmbemu-competition-v1`](https://huggingface.co/datasets/borisbolliet/cmbemu-competition-v1). `cmbemu.load_train()` / `load_test()` download + cache on first use.
- Four files share a schema: `train.npz` (50k, 4 GB), `test.npz` (5k, 380 MB), and `train_small.npz` / `test_small.npz` for dev.
- Seeds: train=202604, test=202605. Do not reuse these seeds when calling `cec.generate_data` to avoid reproducing the held-out test set.

## Conventions worth knowing

- Pass `H0` (not `h`) to classy_szfast. The comment in `emu_competition_data_setup.py:16` notes this matches the emulators and avoids ambiguity when cross-checking with CAMB.
- `get_cmb_cls` returns `ell`-indexed arrays including $\ell = 0, 1$; the likelihood slices from $\ell = 2$. Storage keeps $\ell = 0, 1$ for convenience.
- **`cosmo_model` must be set via `csz.set({"cosmo_model": N})` BEFORE `initialize_classy_szfast()`** — passing it in `params_values_dict` per call is silently ignored. The competition is locked to `cosmo_model=6` (`ede-v2`).
- `chi^2` as defined here is **unit-invariant** — any consistent rescaling of $C_\ell$ (e.g. $D_\ell$ vs $C_\ell$, K² vs μK²) cancels in `Tr` and `log det`.
- Wide-box + CV-limited likelihood → typical off-diagonal $\chi^2 \sim 10^7$; cuts at $\chi^2 < 40$ leave no pairs. Scoring uses **MAE on $\Delta\chi^2$ with no cut**; MSE would be tail-dominated quadratically.
- CMB (TT/TE/EE) and PP are the natural composability units — `chi2_matrix_split` returns them separately; `chi2_mae_blocks` reports the three sub-scores.

## Gotchas

- The `src/` layout means editable installs need `pip install -e .` after any new module (re-run if imports look stale).
- `plot_spectra` was renamed to `plot_spectra_random`; don't resurrect the old name.
- `benchmark_emulator` only *warns* (not errors) when JAX is on GPU unless `strict_cpu=True` — the scoring container sets env vars before the process starts, so the check passes automatically there.
